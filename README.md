## 프로세스

> `fork`와 `execvp`로 셸을 생성하고, 프로세스 계층 구조를 분석했습니다.

### 다룬 내용
- **spawn_shell 구현** — `fork()`로 자식 프로세스를 생성하고, 반환값으로 부모(`pid != 0`)와 자식(`pid == 0`)을 구분. 자식에서는 `execvp()`로 현재 프로세스를 bash 셸로 대체
- **부모 프로세스** — `waitpid()`로 자식 종료를 기다린 뒤, 종료 상태를 출력하고 다시 셸을 생성하도록 설계
- **pstree 분석** — `init` 설정을 `myinit`으로 변경해 최상위 부모 프로세스로 만들고, `myinit → bash`로 이어지는 계층 구조를 확인

### 배운 점
프로세스 생성·대체 메커니즘과 부모-자식 관계가 시스템에서 어떻게 형성되는지 직접 구현하며 이해했다.

## 스레드

> `pthread`로 대규모 합산을 병렬화하고, CPU 점유율을 단일 스레드와 비교했습니다.

### 다룬 내용
- **병렬 합산** — 목표값(50억)을 스레드 수만큼 구간을 나눠 분배. 각 스레드가 맡은 구간을 순회하며 부분합 계산
- **스레드 관리** — `pthread_create`로 스레드별 데이터를 전달, `pthread_join`으로 모든 스레드 완료를 기다린 뒤 부분합을 총합으로 누적. 마지막 스레드는 잔여 범위까지 처리하도록 조건 분기
- **htop 비교** — 단일 스레드는 한 코어만 사용하지만, 다중 스레드는 여러 코어에 작업이 분산되는 것을 확인

### 배운 점
병렬성이 다중 코어 활용을 가능하게 한다는 점을 점유율로 확인했다. 또한 스레드가 같은 프로세스에 속하면서도 htop에서 서로 다른 PID로 보이는 이유(커널이 스레드를 각각의 실행 단위로 관리)를 이해했다.

## 동기화 (Mutex)

> Deadlock을 직접 유발하고, Mutex만으로 예방·회피·탐지/복구를 구현했습니다.

### 다룬 내용
- **Deadlock 유발** — `pthread`로 교착 상태를 만들고, 발생 조건 4가지로 원인을 설명
- **예방(Prevent)** — 두 스레드가 동일한 순서로 자원을 요청하게 해 순환 대기(Circular Wait)를 제거
- **회피(Avoid)** — 자원을 즉시 획득하지 못하면 기존 자원을 반납하고 재시도
- **탐지/복구(Detect/Recover)** — 일정 시간 자원을 획득하지 못하면 타임아웃으로 자원을 반환

### 배운 점
Deadlock의 네 가지 발생 조건과, 각 조건을 깨뜨리는 방식으로 해결책이 나뉜다는 것을 코드로 구현하며 이해했다.

## 동기화 (Semaphore)

> 생산자-소비자 문제와 잠자는 이발사 문제를 결합한 시나리오를 세마포어로 모델링했습니다.

### 시나리오: 드라이브 스루 카페
- **차량 고객(Producer)** — 무작위 간격으로 진입, 대기 차로가 꽉 차면 그냥 지나감(Drive away)
- **주문 직원(Barber)** — 대기 차량이 없으면 휴식, 차량이 오면 주문을 받아 레일로 전송
- **바리스타(Consumer)** — 레일의 주문서를 가져와 음료 제조, 레일이 비면 대기

### 다룬 내용
- 각 객체를 생산자-소비자·잠자는 이발사 모델의 역할에 매핑
- `car_thread` / `register_thread` / `barista_thread`를 세마포어로 동기화
- **데드락 분석** — 레일이 꽉 찬 상태에서 접수 직원이 뮤텍스를 잡고 대기하고, 바리스타는 그 뮤텍스 해제를 기다리는 상호 대기 상황을 설명

### 배운 점
세마포어로 한정된 자원(대기 차로·주문서 레일)을 제어하는 방법과, 동기화 설계가 어긋날 때 발생하는 교착 상태를 구체적인 시나리오로 이해했다.
